Desbloqueo de usuarios y hosts

 

Para desbloquear cuentas de usuario o hosts se debe de hacer desde la terminal de la consola con un usuario con privilegios de root.

 

Desbloqueo de cuentas de usuario

En caso de que un usuario haya provocado demasiados fallos de login y su cuenta se haya bloqueado se puede desbloquear lanzando el siguiente comando desde la terminal de la consola

/opt/qradar/bin/runjava.sh com.ibm.si.security_model.authentication.AuthenticationLockoutCommandLineTool --remove-account <user_account1> <user_account2> <user_account3>

También es posible desbloquear todas las cuentas sin indicar el username de cada una de ellas con el siguiente comando

/opt/qradar/bin/runjava.sh com.ibm.si.security_model.authentication.AuthenticationLockoutCommandLineTool --remove-all-accounts

 

Desbloqueo de hosts

En caso de que un usuario haya fallado el login en demasiadas ocasiones desde una misma IP es posible que esta se bloquee en el SIEM, también es posible que la IP se bloquee si este detecta un ataque por parte de la misma.

Para desbloquear un host lanzar el siguiente comando desde la terminal de la consola especificando la o las IPs a desbloquear

/opt/qradar/bin/runjava.sh com.ibm.si.security_model.authentication.AuthenticationLockoutCommandLineTool --remove-ip <host_IP_address1> <host_IP_address2> <host_IP_address3>

En caso de querer desbloquear todas las IPs bloqueadas, lanzar el siguiente comando

/opt/qradar/bin/runjava.sh com.ibm.si.security_model.authentication.AuthenticationLockoutCommandLineTool --remove-all-ips
