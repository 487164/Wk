Gestión de aplicaciones

 

Para la gestión de aplicaciones a través de la terminal del servidor de QRadar, se puede utilizar el script qappmanager desde la consola de QRadar.

Para ello ejecutar el siguiente comando

sudo /opt/qradar/support/qappmanager

Troubleshooting

En caso de tener problemas con una de las aplicaciones, es posible ver los logs de la misma de dos formas posibles:

  • Desde la consola de qradar en la ruta donde se almacenan los contenedores 
  • Accediendo al propio contenedor

Los logs de los contendores se almacenarán en la consola en la ruta 

/store/docker/volumes/qapp-{ID}/log/

En caso de necesitar acceder al contenedor de la aplicación en cuestión se deberá de obtener el id de aplicación previamente desde el qappmanager en la consola y posteriormente acceder al contenedor.

NOTA: En caso de disponer de un servidor dedicado para el almacenamiento de aplicaciones, el qappmanager se deberá lanzar desde la consola pero el acceso al contenedor se deberá de realizar desde el host primario de APP Node.

Para acceder al contenedor, y una vez hemos obtenido el id de la aplicación desde el qappmanager, obtener el listado de contenedores existentes con el siguiente comando:

docker ps -a

Se ve un ejemplo en la siguiente captura

 

Y acceder al mismo con el siguiente comando:

sudo docker exec -ti {CONTAINER_ID} {SHELL}

Por ejemplo:

sudo docker exec -ti 581461098a8f /bin/bash

Dentro del contenedor, el contenido de la aplicación se encuentra en la ruta

/opt/app-root

Y los logs de la misma pueden verse en la ruta

/opt/app-root/store/log
