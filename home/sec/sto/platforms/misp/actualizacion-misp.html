Actualización de la MISP

 

Para la actualización de la MISP habrá dos posibles formas.

  • Actualización desde la GUI web en caso de que no haya modificaciones en el código
  • Actualización manual desde consola en caso de que se haya modificado alguna parte del código

 

Actualización desde la GUI

Acceder a la GUI con una cuenta con permisos de administrador e ir a la sección

Administration -> Server Settings & Maintenance

Posteriormente en el submenú horizontal acceder al panel de Diagnostics donde se verá una ventana similar a la de la siguiente captura

 

En caso de existir una actualización pendiente aparecerá en esta ventana.

Para actualizar la MISP pinchar sobre el botón “Update MISP” y esperar a que se complete la misma.

Una vez completada la actualización, en la misma ventana de Diagnostics, bajar hasta la sección SQL database status y revisar que no haya cambios pendientes de base de datos.

En caso de haber alguno se deberá acceder por consola a la base de datos y ejecutar manualmente los ALTER que se indiquen en dicha sección.

 

Actualización desde consola

Acceder por ssh a la máquina con un usuario con permisos de root.

Acceder al directorio /var/www/MISP

Y ejecutar los siguientes comandos en orden

sudo -u apache git pull origin 2.4
sudo -u apache git submodule update --init --recursive

Una vez lanzados los comandos acceder desde la GUI a 

Administration -> Server Settings & Maintenance

Posteriormente en el submenú horizontal acceder al panel de Diagnostics y verificar que se encuentra en la última versión disponible.
