Las peticiones de credenciales (Access Request) se asignan a Entitlements a los que tienen acceso usuarios o grupos.

Pueden ser Password Request o Session Request y están asignadas a cuentas locales (accounts) de máquinas (assets).

Para acceder usando cuentas de dominio se crean assets de Active Directory con las cuentas y request que usando cuentas de ese Active Directory se asignen a máquinas.

 

Creación de Usuarios

Rellenar datos, utilizar MFA yubikey

Asignarlo un grupo (Operators, Security, Sysadmins) que le dará acceso a las account

Crear su propio entitlement, las cuentas i50xx, z50xx y n50xx, asignarlas a los assets de jump, al grupo de accounts de jump y las cuentas que puede consultar la API

 

Crear assets

Asignar a partition, asignar cuenta servicio y acceso, rotar, y asignar cuenta a tipo request → Falta instrucciones para preparar asset linux

Crear asset DC, cear asset Active, asignar cuenta de servicio y rotar, importar cuenta de acceso y rotar, crear asset DC sin cuentas, crear request para el DC, asignar cuenta de acceso del active y asignar el asset del dc
