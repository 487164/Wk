En Safeguard for Privileged Passwords (SPP)  Asset Management>Assets se añade la máquina que se va a gestionar:

Windows, servidor independiente o miembro de dominio

En la pestaña General

  • Name: El registro A del DNS de la ip de la interfaz a utilizar (p.ej: itx-sec-poc-gestionaapps-03-mgmt.madrid.espublico.com)
  • Description: El hostname y el cname de la máquina si lo tienen y no son parte del registro A (p.ej. WIN-28CM0AEJLRT ya que a esta máquina no se le ha cambiado el hostname y no tiene cname)

En la pestaña Connection

  • Platform: Windows Server
  • Network Address: IP de la interfaz a utilizar (si tiene más de una, se ha debido configurar el servidor para permitir la conexión por mgmt)
  • Authentication Type: Password
  • Service Account Name: aCdCmN570
  • Password: Teampass (Seguridad>PAM>Windows default password #80287)
  • Click en Test Connection, para comprobar que la cuenta de servicio puede gestionar cuentas en el equipo.
  • Click en OK para guardar

En la pestaña Accounts

En Safeguard for Privileged Passwords (SPP) se dan de alta máquinas (assets) y se les asignan unas credenciales 

Las peticiones de credenciales (Access Request) se asignan a Entitlements a los que tienen acceso usuarios o grupos.

Pueden ser Password Request o Session Request y están asignadas a cuentas locales (accounts) de máquinas (assets).

Para acceder usando cuentas de dominio se crean assets de Active Directory con las cuentas y request que usando cuentas de ese Active Directory se asignen a máquinas.
