Configuración de los Appliances

 

Introducción

Safeguard consta de dos appliances: One Identity Safeguard for Privileged Passwords (PASS o SPP) and One Identity Safeguard for Privileged Sessions (SESS o SPS).

  • PASS: Descubrimiento de activos y cuentas, rotación y gestión de contraseñas y gestión de las peticiones de acceso.
  • SESS: Interceptación transparente o no transparente de protocolos de administración remota, grabación de auditoría y reproducción de sesiones y análisis.

Cuando se utilizan juntos pueden funcionar de dos maneras.

  • En el modo iniciado por PASS, los usuarios solicitan acceso en el portal de PASS y cuando se les otorga acceso, se conectan a la cuenta de destino a través de SESS. Consulta "Uso de SPS con SPP" en la Guía de Administración.
  • En el modo iniciado por SPS, los usuarios se conectan directamente a un servidor de destino, SPS intercepta el tráfico y obtiene las credenciales necesarias de SPP.

Para alta disponibilidad PASS requiere 3 nodos, un primario y dos réplicas (ZGZ, ITX y NBX)

  • Los cambios de configuración solo se pueden realizar en el nodo primario, las peticiones de sesiones y el rotado de contraseñas en ambos.
  • En caso de caída el secundario promociona automáticamente.
  • Si el cluster pierde el quórum entra en modo solo lectura y deshabilita cambios de configuración, se puede configurar si se permiten o no solicitudes de acceso.
  • Si en un nodo PASS un esclavo pasa a maestro dirige automáticamente las conexiones
  • En PASS>Appliance Management>Cluster>Managed Networks se configura a qué redes se llega por cada cluster SESS y se puede modificar en caso de caída.

 

Configuración incial

Se realiza conectando directamente al puerto de MGMT 192.168.1.105

  • El LCD del PASS no muestra información
  • La ILO del PASS no permite conectar por consola (pend. soporte)

 

Contraseñas inciales (se han cambiado y apuntado en TeamPass)

  • Las contraseñas del usuario ADMIN para el acceso por ILO que están en pegatinas en los appliances
  • Pass inicial para la interfaz web y la api admin/Admin123
  • Pass inicial para el acceso por consola root/Admin123

 

Se configuran las interfaces de red config/basic/network en SESS y /appliance-management/appliance?section=networking en PASS y se actualizan en

https://docs.google.com/spreadsheets/d/1sfXXKECd3blN6z9jFFGEL5KYYBEyBCfTCTZpqkXDcxI

  • Las interfaces X0 se utilizan para SVCS (cluster entre máquinas, acceso web de usuarios y administradores)
  • Las interfaces X1, la Physical interface 3 en SESS, para MGMT (conectar hacia servidores, GW por defecto.)
  • Las interfaces IPMI, para INFRA

En SESS es necesario cambiar también la Routing table en /config/basic/network para que utilice el GW de MGMT

 

Se configuran los nombres de máquina en SESS config/basic/network y en PASS appliance-management/appliance?section=applianceInformation
