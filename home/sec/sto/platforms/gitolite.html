Gitolite

 

Actualmente se utiliza la herramienta gitolite para el almacenamiento de código en repositorios siguiendo el formato de git.

Para el acceso a los diferentes repositorios está configurado el acceso tanto por ssh como por http/s a través del servidor itx-sec-cm-01-svcs.madrid.espublico.com aunque es posible acceder con el CNAME cm.security.espublico.com

 

Se encuentran configurados los siguientes repositorios que se describirán en páginas individuales para cada uno de ellos:

  • certificates → Almacenamiento de certificados ssl
  • gitolite-admin → Utilizado para el almacenamiento de la configuración de gitolite
  • Security-monitoring → Almacenamiento de scripts y aplicaciones utilizadas en los departamentos de Security-monitoring
  • sshkeypairs → Almacenamiento de claves ssh utilizadas en diferentes servicios
  • Threat-intelligence → Almacenamiento de scripts y aplicaciones utilizadas en los departamentos de Threat-intelligence
  • Threat-hunting → Almacenamiento de scripts y aplicaciones utilizadas en los departamentos de Threat-hunting
  • Vulnerability-mgmt → Almacenamiento de scripts y aplicaciones utilizadas en los departamentos de Vulnerability-mgmt
  • wikidata → Backup de las páginas existentes en esta wiki
