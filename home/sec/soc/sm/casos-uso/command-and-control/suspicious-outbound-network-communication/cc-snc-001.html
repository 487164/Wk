CC-SNC-001[MEDIA][MAIL] Envio de mail a servidor malicioso.

Descripción

Este caso de uso identifica un usuario a enviado un correo a un servidor malicioso .

  • Nombre: Envio de mail a servidor malicioso.
  • Criticidad: Alta
  • Tiempo de Atención: 30 minutos
  • Tecnologías utilizadas: Logs locales

MITRE

  • TA0011 – Command and Control

What to do

Una vez identificado el dominio alojado en el servidor malicioso y la ip, lo primero que debemos de realizar es, escanearlos con las diferentes herramientas (x-Force, VirusTotal, Cisco Talos, e.t.c) tanto del dominio como la IP, si es cierto que el dominio o la IP al cuál se ha enviado el correo es malicioso, deberíamos de identificar el porque, si es por SPAM, por escaneo de puertos e.t.c. A continuación vemos de donde proviene ese dominio (si es un dominio por ejemplo de un ayuntamiento, universidad, e.t.c). Si tras este análisis detectamos que el dominio es legítimo, o por ejemplo fue detectado como SPAM, pero hace tiempo, lo descartaríamos y cerraremos la alerta como FP.

En caso de que detectemos que el dominio o la IP son maliciosas, tendríamos que verificar la legitimidad del dominio o la IP, miraremos desde el exchange, quién ha enviado el correo, que tipo de documentación ha enviado, qué tipo de malware ha detectado. En caso que no encontrar nada de información en el exchange porque por el servidor que pasa la información es “relay=nginx*” tendríamos que escalar (Véase punto 5) ofreciendo los siguientes datos:

  • La ID del evento. (EJ: 33C1A8A6A85)
  • La fecha.
  • El servidor. (EJ: relay=nginx-01.madrid.espublico.com[213.4.82.201])

Si consideramos que la documentación enviada y el dominio no son maliciosos lo cerramos como FP. Si consideramos que la documentación enviada era sensible, o el dominio era malicioso, pero se ha detenido lo cerramos como PC y en caso de que detectemos que se ha enviado documentación sensible o el dominio era malicioso y no se ha detenido, escalaremos (Véase punto 5).

Notas de cierre

Puntos mínimos que debe de tener un cierre

  • nombre del dominio o IP por el cuál ha saltado la alerta.
  • información del escaneo del dominio o IP
  • en caso de ser posible, obtener el usuario de quien envió el correo.
  • documentación, de la fecha, la ID y el servidor de donde se manda el correo
  • resumen del análisis

Matriz de escalado

En horario laboral: Atiende N1 y escala a N2

Fuera del horario Atiende N0 y escala a Guardia
