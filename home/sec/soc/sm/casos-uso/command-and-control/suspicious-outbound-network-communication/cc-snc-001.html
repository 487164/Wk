CC-SNC-004 [BAJA] [ESFIRMA] [NTP] Error de sincronizacion de ntp en servidores m0x de esFirma

Descripción

Esta alerta se activa cuando se generan eventos con el QID 2000131 (mensaje ntpstat) que no son detectados por el Sistema Local, y cuando dicho mensaje contiene alguna sincronización, y un día después, fueron detectados por:
Linux OS @mad-esfirma-m01-svcs.pki.esfirma.com
Linux OS @mad-esfirma-m02-svcs.pki.esfirma.com

  • Nombre: Error de sincronización de ntp en servidores m0x de esFirma
  • Criticidad: Baja
  • Tiempo de Atención: 180 minutos
  • Tecnologías utilizadas: Logs locales

MITRE

Esta regla no aplica a Mitre ATT&CK, ya que es una regla para monitorizar un fallo en la configuración o funcionamiento del servicio NTP de los servidores. 

What to do

Una vez detectada la alerta hay que crear un Ticket en RT  indicando la incidencia para que los administradores tomen medidas correctivas.

El Ticket tiene que ser creado en la cola de SO Problemas indicando la ID de la alerta, la  [URL Referrer (custom)] y [URL Query String (custom))] 

Notas de cierre

Puntos mínimos que debe de tener un cierre

  • Nº de Ticket en RT
  • Breve descripción de la alerta
  • Máquina afectada.

Matriz de escalado

En horario laboral: Atiende en primera instancia N1 y si es necesario escalar, pasaría a N2.

Fuera del horario Atiende el técnico de 24/7 que se encuentre y escalara después del análisis inicia a Guardia si fuera necesario.

Reglas de Caso de Uso

Campos Necesarios

Los campos mínimos que deben seleccionarse para llevar a cabo la investigación:

  • Start Time
  • Event Name
  • Source IP
