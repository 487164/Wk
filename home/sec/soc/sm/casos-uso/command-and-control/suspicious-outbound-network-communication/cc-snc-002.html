CC-SNC-002 [MEDIA] [MULTI] Comunicacion hacia dirección ip maliciosa

Descripción

Esta alerta se activa cuando se detecta una comunicación hacia una IP potencialmente peligrosa, la cual es categorizada como tal por X-Force o por otra herramienta de referencia.  De manera que nos alerta de las comunicaciones de local a remoto, evento es detectado por el sistema local, en “Action (custom)” y “rule_staus (custom)” es “DENY”, la IP de destino no es “195.77.90.84” y el puerto de destino no es el “80” ni el “443”. 

  • Nombre: Comunicación hacia dirección IP maliciosa.
  • Criticidad: Alta
  • Tiempo de Atención: 30 minutos
  • Tecnologías utilizadas: Logs locales

MITRE

  • TA0011 – Command and Control

What to do

Una vez detectada la alerta hay que revisar al usuario inicia la alerta así mismo como la IP de destino para ver si es maliciosa, para ello se deberán usar varias herramientas incluyendo la de X-Force, ya que X-Force conserva los delitos de una IP durante años, por lo que tendremos que sumar los resultados de las otras herramientas para verificar el tipo de actividad maliciosa que podría tener dicha IP, analizando los datos y las fechas de los últimos incidentes asignados a dicha IP se puede determinar si es un Falso_Positivo, o un Positivo_Contenido.

Notas de cierre

Puntos mínimos que debe de tener un cierre

  • Usuario de origen y un análisis de su actividad
  • Análisis de la IP que ha provocado el salto de la alerta
  • Control de la actividad de dicha IP en las últimas semanas. 
  • Resumen del análisis

Matriz de escalado

En horario laboral: Atiende en primera instancia N1 y si es necesario escalar, pasaría a N2.

Fuera del horario Atiende el técnico de 24/7 que se encuentre y escalara después del análisis inicia a Guardia si fuera necesario.
