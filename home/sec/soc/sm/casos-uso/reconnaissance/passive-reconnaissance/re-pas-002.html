RE-PAS-002 [BAJA] [ESPUBLICO] [DNS] Intento de transferencia de zona

Descripción

Este caso de uso identifica un intento de transferencia de zona DNS a nuestra infraestructura.

  • Nombre: Transferencia de zona
  • Criticidad: Baja
  • Tiempo de Atención: 120 minutos
  • Tecnologías utilizadas: ISC BIND

MITRE

  • TA0043 – Reconnaissance
  • T1596 - Search Open Technical Databases
    • T1596.001 - DNS/Passive DNS

What to do

Analizar la IP desde la cual están intentando realizar la transferencia DNS, dado que según pruebas realizadas la transferencia de zona está configurada solo para que ciertos servidores puedan hacerla no hay riesgo de que se produzca satisfactoriamente. Si por algún fallo se pudiera hacer, saltaría la ofensa RE-PAS-001 [MEDIA] [ESPUBLICO] [DNS] Transferencia de zona exitosa con la cual habría que seguir el procedimiento de dicha ofensa.

Notas de cierre

Puntos mínimos que debe de tener un cierre

  • Start Time
  • Event Name
  • Source IP
  • Destination IP
  • Resumen del análisis

Matriz de escalado

Aunque la matriz de escalado debería ser la misma para todos los casos de uso, puede haber casos que por falta de capacidades, por dependencias de otros servicio o incluso por que así se acuerde para un caso concreto, se debe definir la matriz de escalado.
