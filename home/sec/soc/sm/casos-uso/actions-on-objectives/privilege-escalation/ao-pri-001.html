AO-PRI-001 [MEDIA] [ESPUBLICO] [CMDB] GLPI Impersonation

Descripción

Este caso de uso identifica la elevación de privilegios y suplantación de identidad del usuario GLPI en la CMBD, el cual es un super admin.

  • Nombre: GLPI Impersonation
  • Criticidad: Media
  • Tiempo de Atención: 90 minutos
  • Tecnologías utilizadas: CMDB

MITRE

  • TA0001 - Initial Access
  • TA0004 - Privilege Escalation

What to do

Una vez identificada una suplantación de identidad del usuario GLPI, habrá que comprobar quién ha sido el usuario de origen que ha hecho esa elevación:

username -> starts impersonating user -> glpi

Nadie debería hacer uso del usuario glpi, ya que es un super admin, por lo que habría que consultar si hay una razón justificada para hacer uso del mismo.

Cada usuario tiene su propio usuario nominal en la cmdb, por lo que deben hacer uso de los mismos para tener controlada la actividad.

Notas de cierre

Puntos mínimos que debe de tener un cierre

  • Fecha / Hora de la suplantación de glpi
  • Usuario de origen
  • Resumen del análisis

Matriz de escalado

Aunque la matriz de escalado debería ser la misma para todos los casos de uso, puede haber casos que por falta de capacidades, por dependencias de otros servicio o incluso por que así se acuerde para un caso concreto, se debe definir la matriz de escalado.
