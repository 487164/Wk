AO-ACC-030 [MEDIA] [ESPUBLICO] [IPAM] Cambio de MFA en PAM detectado

Descripción

Este caso de uso identifica el cambio del MFA de un usuario dentro de Oneidentity Safeguard

  • Nombre: Cambio de MFA en PAM detectado
  • Criticidad: Media
  • Tiempo de Atención: 90 minutos
  • Tecnologías utilizadas: Oneidentity Safefuard

MITRE

  • TA0001 - Initial Access

What to do

Una vez identificado el usuario que ha realizado el cambio sobre la llave Fido del SPP se debe pregunta que ha hecho y el motivo ya que el cambio de la key a otro tipo de token no se debe hacer y por ahora no es posible bloquear esto.

Notas de cierre

Puntos mínimos que debe de tener un cierre

  • Fecha / Hora del login del cambio
  • Datos del usuario
  • Motivo del cambio
  • Resumen del análisis

Matriz de escalado

Aunque la matriz de escalado debería ser la misma para todos los casos de uso, puede haber casos que por falta de capacidades, por dependencias de otros servicio o incluso por que así se acuerde para un caso concreto, se debe definir la matriz de escalado.

En horario laboral: Atiende N1 y escala a N2

Fuera del horario Atiende N0 y escala a Guardia

Reglas de caso de uso

Apply AO-ACC-030 [MEDIA] [ESPUBLICO] [IPAM] Cambio de MFA en PAM detectado on events which are detected by the Local system
and when the event QID is one of the following (1012000040) Fido 2 Authenticator Updated

Campos Necesarios

Los campos mínimos que deben seleccionarse para llevar a cabo la investigación:

  • Source IP
  • Username
  • Motivo
  • Resolución
  • Time

Cierre por defecto

Nota de cierre que se debería de rellenar a la hora de cerrar una alerta:

Falso positivo:

  • Se ha detectado que desde la [SourceIP]  se ha hecho un cambio en la token del usuario [Username] a las [Time]. Tras ponerse el contacto con el usuario nos ha indicado que es por [Motivo] por lo que finalmente [Resolución]. Se cierra como FP.
