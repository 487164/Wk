AO-ACC-019 [MEDIA] [ESPUBLICO] [MULTI] Posible viaje imposible detectado

Descripción

Este caso de uso recopila información como dirección GPS, dirección IP o el dispositivo del usuario para determinar la localización de un usuario y determinar si su comportamiento es físicamente posible.

  • Nombre: Posible viaje imposible detectado
  • Criticidad: Media
  • Tiempo de Atención: 90 minutos
  • Tecnologías utilizadas: QRadar

MITRE

TA0006 – Credential Access.

TA0001 - Initial Access.

What to do

Se debe identificar qué usuario está generando los eventos, y desde qué ubicaciones. Se tendrá que analizar si se trata de un fallo de la geolocalización de QRadar, el usuario está utilizando un VPN, o por el contrario se ha producido un incidente de seguridad y están utilizando el usuario identificado de forma no controlada

Notas de cierre

Puntos mínimos que debe de tener un cierre

  • Posible cuenta comprometida
  • Fecha / Hora de los Login sospechosos
  • Direcciones IP donde se han producido los Login
  • Geolocalización de dichas IP
  • Acciones posteriores al Login
  • Resumen del análisis

Matriz de escalado

Aunque la matriz de escalado debería ser la misma para todos los casos de uso, puede haber casos que por falta de capacidades, por dependencias de otros servicio o incluso por que así se acuerde para un caso concreto, se debe definir la matriz de escalado.

En horario laboral: Atiende N1 y escala a N2

Fuera del horario Atiende N0 y escala a Guardia
