AO-ACC-009 [MEDIA] [ESFIRMA] [EJBCA] Autenticación Web Admin

Descripción

Este caso de uso identifica un acceso válido en el panel web de la ejbca no contemplado con un usuario no administrador de ejbca.

  • Nombre: Autenticación Web Admin
  • Criticidad: Media
  • Tiempo de Atención: 90 minutos
  • Tecnologías utilizadas: PKI Ejbca

MITRE

  • TA0006 - Credential Access.
  • TA0001 - Initial Access
  • TA0004 - Privilege Escalation

What to do

Una vez identificado un posible acceso a Web Admin, habrá que identificar qué usuario ha sido y desde qué dirección IP.

Se deberá poner en contacto con el usuario para verificar si el inicio de sesión es controlado, y revisar los movimientos posteriores al login. 

En caso de que el usuario confirme que es responsable de dichos login, lo definiríamos como FP descartando un incidente de seguridad y proporcionando la nota de cierre correspondiente con la investigación realizada.

Notas de cierre

Puntos mínimos que debe de tener un cierre

  • Posible cuenta comprometida
  • Fecha / Hora de los Login sospechosos
  • Dirección IP donde se han producido los Login
  • Acciones posteriores al Login
  • Resumen del análisis

Matriz de escalado

Aunque la matriz de escalado debería ser la misma para todos los casos de uso, puede haber casos que por falta de capacidades, por dependencias de otros servicio o incluso por que así se acuerde para un caso concreto, se debe definir la matriz de escalado.
