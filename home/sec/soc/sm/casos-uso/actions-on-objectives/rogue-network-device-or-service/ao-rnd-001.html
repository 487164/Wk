AO-RND-001 [BAJA] [ESPUBLICO] [DHCP] DHCP Spoofing

Descripción

Este caso de uso identifica un posible caso de DHCP Spoofing en la red 2. En esta red el único dhcp server que hay es el proxy 195.77.90.109

  • Nombre: DHCP Spoofing
  • Criticidad: Baja
  • Tiempo de Atención: 180 minutos
  • Tecnologías utilizadas: Linux DHCP Server @ proxy.auloce.local

 

MITRE

  • TA0006 - Credential Access
  • T1557 - Adversary in the middle
    • T1557.003 - DHCP Spoofing

What to do

Una vez detectado un evento de DHCP ACK, donde un servidor esté contestando a una request IP de un equipo en la red 192.168.2.0/24, deberá verificarse si no ha sido un fallo en la lógica de la regla. Solo deberían generarse DHCP ACK desde:

  • Linux DHCP Server @ proxy.auloce.local (195.77.90.109) 

En caso de verse desde otra dirección IP en la red 192.168.2.0/24, deberá consultarse a sistemas si se ha cambiado la dirección IP del proxy o si se ha levantado otro servidor dhcp por alguna razón en especial. 

Si nadie sabe nada sobre este DHCP Server levantado, deberá investigarse donde se ha levantado, investigando la IP de origen. También revisar el equipo al que se le ha asignado la dirección IP de este servidor, pudiendo investigar en los logs del QRadar la dirección MAC y el nombre de equipo para identificarlo. 

En caso de que Sistemas sea consciente, lo definiremos como Falso_Positivo, descartando un incidente de seguridad y proporcionando la nota de cierre correspondiente con la investigación realizada.

Puntos mínimos que debe de tener un cierre
