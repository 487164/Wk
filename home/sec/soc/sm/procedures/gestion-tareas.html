Gestión de tareas

 

El presente documento detalla los flujos de tareas para el servicio de Security Monitoring

 

Herramientas de ticketing

La herramienta elegida para la gestión del servicio es RT, disponible en https://soporte.iubaris.com

 

Colas asignadas al servicio

El servicio dispone de las siguientes colas y características:

  • Nombre: SD-SOC-Security_Monitoring
  • Correo: rt_soc_sm@soporte.espublico.it
  • Grupo de Permisos:
    • Acceso y comentarios: Seguridad
    • Gestión de la cola: SD-SOC-Security_monitoring-Operadores
  • TAG: [SOC_SM]

 

Generación de tickets

Los tickets se generarán directamente desde la plataforma o enviando correo al buzón asignado a la cola.

El asunto debe cumplir la siguiente nomenclatura:

[Tipo de tarea]-[Periodicidad]:Descripción

Para las tareas recurrentes se añadirá al final de la descripción la fecha en formato dd-MM-yyyy.

Ejemplo de solicitud: [REP]: Nuevo report para controles VPN

Ejemplo de tareas recurrentes: [REV]-[D]: Revisión de salud de los event collector 10-02-2022

Cualquier solicitud que no se ajuste a los TAGs definidos podrán ser abiertos con el TAG [OTRAS].

 

Tags para tickets

  • Tipo de tarea(Obligatorio):
    • Investigación: [INV]
    • Informe: [REP]
    • KPIs: [KPI]
    • Nuevo caso de uso: [NUC]
    • Documentación: [DOC]
    • Otras: [OTRAS]
  • Periodicidad (Solo para tareas recurrentes):
    • Diario:[D]
    • Semanal:[W]
    • Quincenal: [B]
    • Mensual: [M]
    • Trimestral:[3M]
    • Cuatrimestral:[4M]
    • Semestral:[6M]
    • Anual:[Y]
